Parallelism(//-ism)
































Observations:















Looking at the running times of different board sizes, we notice a well-known property of NP-complete problems; the time seems to grow exponentially as n increases. Running times for small input size are OK, but it gets intractable for large input sizes(Board sizes more than 20 take hours to find a minimum-cost solution and days for the 28x28 board!!!)















Parallelism improves(increases) as n increases. This is expected as larger board sizes are more parallelizable, thus minimizing processors idle times.















Linear speedup is almost achieved starting from board size n=20; which means that the program is most parallelizable with large board sizes(n>=20)















The efficiency increases with the board size and the no. of processors.















The above statistics strongly suggest that this problem is NP-Complete(can be easily verified in polynomial time but finding a solution takes exponential time). Since the typical N-Queens problem is NP-complete; therefore the weighted board version of N-Queens is certainly NP-complete too(can be proved by restriction from weighted Nqueens to typical Nqueens by putting equal weights for all positions).
